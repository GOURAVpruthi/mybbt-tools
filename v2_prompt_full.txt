**Normalization steps:**
1. Uppercase everything
2. Remove: `.`, `,`, `/`, `&`, `-`, `_`, `(`, `)`
3. Collapse multiple spaces вЖТ single space
4. Strip suffixes: LIMITED, LTD, PRIVATE, PVT, COMPANY, INC, CORPORATION, CORP, BANK (standalone only)
5. Apply alias dictionary (SBI вЖТ STATE BANK OF INDIA etc.)
6. Use resulting key for grouping

**Critical Rule вАФ Bank Match Amount Logic:**
- Bank entries: Tax Head MUST match (IGST vs CGST+SGST) вАФ no cross-head bank match
- State MUST match for bank entries
- Amount variance in bank match is NOT acceptable (unlike regular invoices)
- If PR total вЙ† 2B total for a bank group вЖТ do NOT force-match
- Instead вЖТ **suggest split:** "PR can be split as вВєX + вВєY to match 2B entry of вВєX; remaining вВєY is excess in PR"

## STEP 6 вАФ POST-RUN AMOUNT MISMATCH RE-ANALYSIS

**This is a critical new step.** After full reco is done, look at all "Amount Mismatch" rows and re-examine.

### Why This Is Needed:

Consider this scenario:
```
PR has:  Invoice 11 вЖТ вВє1000 tax  (10 line items)
         Invoice 11 вЖТ вВє100 tax   (entered as "011" by mistake)
2B has:  Invoice 11 вЖТ вВє1100 tax

Simple match: PR вВє1000 vs 2B вВє1100 вЖТ Amount Mismatch!
But: If we combine PR "11" (вВє1000) + PR "011" (вВє100) вЖТ вВє1100 = 2B вВє1100 вЖТ PERFECT MATCH!
```

### Re-Analysis Algorithm:

For every pair/group marked "Amount Mismatch":
1. Identify all PR rows with same GSTIN/PAN that are currently in unmatched pool OR in mismatch bucket
2. Try all combinations (up to max_combo) to see if any subset sums to the 2B amount
3. Also try: can 2B side have multiple rows that sum to PR amount?
4. Consider: same Invoice No with leading zero variant (11 вЖФ 011 вЖФ 0011)
5. Consider: same Invoice No across different months/dates (split across FY reco)

**Cross-FY consideration:**
```
Example: 2-year combined reco
  PR FY24: Invoice 11 вЖТ вВє500
  PR FY25: Invoice 11 вЖТ вВє600
  2B:      Invoice 11 вЖТ вВє500 (FY24 date)
           Invoice 11 вЖТ вВє600 (FY25 date)
  
  вЖТ Match each by FY separately (date month determines FY)
  вЖТ Do NOT combine cross-FY amounts
  вЖТ Remark: "Cross-FY Invoice вАФ Matched by FY-wise date"
```

**Output of Re-Analysis:**
- Rows that now match вЖТ move to Matched sheet with remark: `Re-Analysis Match вАФ Combination [INV A + INV B]`
- Rows still mismatched вЖТ keep in Amount Mismatch sheet
- Add column: `Re_Analysis_Attempted: Yes/No`

## AMOUNT MISMATCH вАФ DEEP ANALYSIS PROTOCOL
